## Key Decision: Honest Positioning

**Original claim:** "v0.2 - Feature complete for full OpenCypher"
**Revised positioning:** "v0.2 - Core Cypher-compatible for common graph operations"

After OpenCypher TCK analysis, v0.2 represents ~25% compliance (950/3,837 scenarios), not "full OpenCypher." The revised plan:
- **Provides better breadth** - 9 diverse features vs. 5 (with 1 very complex)
- **Sets honest expectations** - Clear about gaps and limitations
- **Establishes roadmap** - Multi-version path to 70-75% compliance (v1.0)
- **Builds trust** - Transparent about what GraphForge is and isn't

GraphForge is **not** competing with Neo4j as a production database. It's an **embedded, SQLite-like graph query engine** for Python data science workflows â€” and that's a valuable, unique position.
